## Что такое сниппеты

Сниппеты - это шаблоны кода, которые можно быстро вставить в редактор. Они содержат placeholder'ы (заполнители) для навигации и редактирования.

### Пример

При вводе `for` и нажатии `Tab` вставляется:

```proxima
for ${1:i} in ${2:range}
    ${3:// code}
end
```

Курсор автоматически переходит к ${1:i}, где вы можете ввести имя переменной.

## Синтаксис placeholder'ов

Базовый синтаксис

``` proxima
${id:default_text}
```

- id - номер placeholder'а (определяет порядок навигации)
- default_text - текст по умолчанию

## Примеры

``` proxima
${1:name}           // Placeholder с именем
${2:100}            // Placeholder с числом
${3:// comment}     // Placeholder с комментарием
${4}                // Placeholder без текста (пустой)
```

## Связанные placeholder'ы

Placeholder'ы с одинаковым id синхронизируются:

``` proxima
class ${1:ClassName}
    ${1:ClassName}() : constructor
    end
end
```

При изменении первого ${1:ClassName}, второй автоматически обновляется.

## Выбор из списка (Choice)

``` proxima
${1|public,protected,private|}
```

Пользователь может выбрать один из вариантов.

## Вложенные placeholder'ы

``` proxima
function ${1:name}(${2:args}) : ${3|void,int32,double,string|}
    ${4:// code}
end
```

## Навигация по сниппетам

Клавиши навигации

| Клавиша | Действие |
| ------- | -------- |
| Tab | Следующий placeholder |
| Shift+Tab | Предыдущий placeholder |
| Escape | Выйти из режима сниппета |
| Enter | Принять текущий placeholder |

## Индикаторы

Синяя подсветка - активный placeholder
Серая подсветка - неактивный placeholder
Tooltip - информация о текущем placeholder

## Поведение

1. После вставки сниппета курсор помещается в первый placeholder
2. Текст placeholder'а выделен для быстрого редактирования
3. Нажатие Tab переходит к следующему placeholder
4. При достижении последнего placeholder, Tab завершает сессию
5. Shift+Tab движется в обратном направлении

## Создание пользовательских сниппетов

Формат файла сниппетов

[
    {
        "trigger": "myfunc",
        "name": "My Function",
        "description": "Custom function template",
        "content": "function ${1:name}(${2:args}) : ${3:void}\n    ${4:// code}\nend",
        "language": "proxima",
        "scopes": ["source.proxima"],
        "relevance": 100
    }
]

## Поля

| Поле | Описание |
| ---- | -------- |
| trigger | Триггер для активации |
| name | Название сниппета |
| description | Описание для autocomplete |
| content | Содержимое с placeholder'ами |
| language | Язык (proxima) |
| scopes | Области применения |
| relevance | Приоритет (0-100) |

## Диалог управления

- Предпросмотр

## Горячие клавиши

| Действие | Клавиши |
| -------- | ------- |
| Вставить сниппет | Tab (после триггера) |
| Следующий placeholder | Tab |
| Предыдущий placeholder | Shift+Tab |

##Примеры использования

### Пример 1: Быстрое создание функции

1. Введите func
2. Нажмите Tab
3. Введите имя функции (первый placeholder)
4. Нажмите Tab
5. Введите аргументы (второй placeholder)
6. Нажмите Tab
7. Введите тип возврата (третий placeholder)
8. Нажмите Tab
9. Напишите код тела функции
10. Нажмите Tab для выхода

### Пример 2: Создание класса

1. Введите class
2. Нажмите Tab
3. Введите имя класса
4. Нажмите Tab для перехода к конструктору
5. Заполните конструктор
6. Нажмите Tab для перехода к методам
7. Добавьте методы
8. Нажмите Tab для выхода

### Пример 3: Цикл с переменной

1. Введите for
2. Нажмите Tab
3. Введите имя переменной (например, i)
4. Нажмите Tab
5. Введите диапазон (например, 1:100)
6. Нажмите Tab
7. Напишите тело цикла
8. Нажмите Tab для выхода

## Советы

- Используйте осмысленные триггеры - короткие и запоминающиеся
- Группируйте placeholder'ы - одинаковые id для синхронизации
- Добавляйте комментарии - в placeholder'ы для подсказок
- Тестируйте сниппеты - убедитесь, что навигация работает
- Делитесь сниппетами - экспортируйте для команды
