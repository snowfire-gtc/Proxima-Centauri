## Введение

Centauri IDE - это интегрированная среда разработки для языка программирования Proxima. IDE предоставляет все необходимые инструменты для написания, отладки и визуализации кода.

### Основные возможности

- Редактор кода с подсветкой синтаксиса
- Визуальный отладчик с точками останова
- Визуализация векторов, матриц и слоёв
- Интеграция с Git
- Многопользовательское редактирование
- Авто-сохранение
- Интеграция с LLM (AI-помощник)

## Начало работы

### Запуск IDE

``` bash
# Linux/Mac
centauri

# Windows
centauri.exe
```

## Командная строка

``` bash
# Открыть проект
centauri --project /path/to/project

# Открыть файл
centauri --file /path/to/file.prx

# Тёмная тема
centauri --theme dark

# Подробный лог
centauri --verbose
```

## Создание проекта

### Новый проект

1. Меню Файл → Новый проект
2. Выберите расположение проекта
3. Введите имя проекта
4. Нажмите Создать

### Структура проекта

```
project_name/
├── src/                    # Исходные тексты
│   ├── main.prx           # Точка входа
│   └── module.prx
├── rules/                  # Конфигурация
│   ├── build.rules
│   ├── design.rules
│   └── ui.rules
├── assets/                 # Ресурсы
├── autosave/              # Авто-сохранения
└── manifest               # Манифест проекта
```

### Открытие проекта

1. Меню Файл → Открыть проект
2. Выберите папку проекта
3. Нажмите Открыть

## Редактирование кода

### Открытие файлов

- Двойной клик на файле в дереве проекта
- Меню Файл → Открыть файл
- Горячая клавиша: Ctrl+O

### Навигация

| Действие | Способ |
| :------- | ------ |
| Переключение между файлами | Табы, Ctrl+Tab |
| Поиск файла | Ctrl+P |
| Переход к строке | Ctrl+G |
| Последниe файлы | Ctrl+1..Ctrl+9 |

### Режимы отображения

Редактор поддерживает несколько режимов подсветки:

1. Стандартный - подсветка синтаксиса
2. Типы - подсветка типов переменных
3. Размерности - подсветка размерностей массивов
4. Время - время выполнения кода
5. Частота - частота выполнения
6. Git Diff - изменения vs репозиторий
7. Автор - автор правки
8. Старость - время последней правки
Переключение: панель инструментов редактора → кнопка режима

### Авто-сохранение

Файлы автоматически сохраняются каждые 5 минут в папку autosave/.

- Редактируемый файл сохраняется каждые 5 минут
- Если файл не редактировался 5+ минут - однократное сохранение
- Папка: autosave/src/ (зеркальная структура src/)

## Отладка

### Запуск программы

| Режим | Описание | Горячая клавиша |
| ----- | -------- | --------------- |
| Release | Минимальная нагрузка | Ctrl+F5 |
| Debug | Полная отладка | F5 |
| До точки останова | Остановка на breakpoint | Shift+F5 |
| До курсора | Выполнение до курсора | Ctrl+F10 |
| До выхода из метода | До return метода | Ctrl+Shift+F10 |
| До первого метода | До первого вызова | Alt+F5 |

### Точки останова

Создание

- Клик на поле слева от номера строки
- Горячая клавиша: F9
- Меню Выполнение → Toggle Breakpoint

Типы точек останова

1. Безусловная - остановка всегда
2. Условная - остановка при выполнении условия
3. Временная - остановка N раз

Управление

``` Proxima
// В коде
breakpoint;              // Безусловная
breakpoint(count=5);     // Временная (5 итераций)

// Условная
if x > 100
    breakpoint;
end
```

### Шаги отладки

| Действие | Описание | Горячая клавиша |
| -------- | -------- | --------------- |
| Step Over | Шаг через метод | F10 |
| Step Into | Шаг внутрь метода | F11 |
| Step Out | Шаг из метода | Shift+F11 |
| Continue | Продолжить | F5 |
| Pause | Пауза | Ctrl+Pause |
| Stop | Остановить | Shift+F5 |

### Инспектор переменных

Во время паузы можно просматривать и изменять переменные:

- Дерево переменных (панель отладки)
- Двойной клик для редактирования значения
- Правый клик → Visualize для визуализации

## Визуализация данных

### Векторы

Открытие:

- Правый клик на переменной → Visualize
- Автоматически при остановке на векторе

Инструменты:

- Zoom In/Out (+/-)
- Pan (перетаскивание)
- Find Peaks/Valleys
- Выделение диапазона
- Копирование данных
- Сохранение как CSV/Image

Режимы:

- График (линейный)
- Точки
- Комбинированный

### Матрицы

Режимы отображения:

- Таблица (числовые значения)
- Изображение (тепловая карта)
- Heatmap (с палитрой)

Инструменты

- Zoom/Pan
- Выбор палитры (Viridis, Plasma, etc.)
- Наложение матриц
- Выделение диапазона
- Обрезка матрицы
- Копирование данных

Настройки

- Показать значения (при большом zoom)
- Показать сетку
- Показать оси
- Показать colorbar

### Слои (3D)

Режимы

- Wireframe (каркас)
- Solid (твёрдое тело)
- Volume (объёмное)

Управление

- Вращение (Ctrl + перетаскивание)
- Zoom (колесо мыши)
- Pan (Shift + перетаскивание)
- Выбор сечения (слайдер)

Инструменты

- Сохранение сечения
- Экспорт данных
- Копирование данных

### Коллекции

Отображение

- Таблица с заголовками
- Сортировка по столбцам
- Фильтрация

Операции

- Добавление/удаление строк
- Добавление/удаление столбцов
- Сортировка
- Фильтрация
- Экспорт (CSV, JSON, Excel)

## Работа с Git

### Инициализация

1. Меню Проект → Git → Initialize
2. Или через консоль: git init

### Основные операции

| Операция | Меню | Горячая клавиша |
| -------- | ---- | --------------- |
| Status | Project → Git → Status | - |
| Add | Project → Git → Add | - |
| Commit | Project → Git → Commit | Ctrl+Alt+C |
| Push | Project → Git → Push | Ctrl+Alt+P |
| Pull | Project → Git → Pull | Ctrl+Alt+L |

### Сравнение с репозиторием

- Режим редактора: Git Diff
- Показывает изменения vs выбранной ветки
- Цветовая индикация:
🟢 Зелёный - добавленные строки
🔴 Красный - удалённые строки
🟡 Жёлтый - изменённые строки

### Ветки

- Просмотр веток: Project → Git → Branches
- Создание ветки: Project → Git → Create Branch
- Переключение: Project → Git → Checkout

## Многопользовательская работа

### Создание сессии

1. Меню Генерация → Collaboration → Host Session
2. Укажите порт (по умолчанию 8080)
3. Отправьте участникам: адрес и порт

### Подключение к сессии

1. Меню Генерация → Collaboration → Join Session
2. Введите адрес хоста и порт
3. При необходимости введите пароль

### Возможности

- Синхронное редактирование - несколько пользователей редактируют один файл
- Курсоры пользователей - отображение курсоров других участников
- Выделения - отображение выделений других участников
- Блокировка файлов - исключение конфликтов
- История правок - учёт всех изменений
- Откат изменений - частичный или полный откат

### Управление правами

Хост сессии может:

- Откатывать изменения пользователей
- Блокировать участников
- Завершать сессию
