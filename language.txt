Давай с помощью LLVM разработаем Matlab-совместимый язык программирования "Проксима" ("Proxima"). Внимательно изучи описанные требования и положения, глубоко продумай и сформируй подробное и обязательно полное описание языка для последующей разработки LLM-моделью языка программирования и сопутствующей IDE, названной "Кентавра" ("Centauri").
У тебя много времени - никуда не торопись.
Особенности языка:
1. Общее.
	1.1. Название: Proxima.
	1.2. Лицензия: GPLv3.
	1.3. Исходные тексты имеют расширение ".prx" и хранятся в текстовом формате.
	1.4. Лексер и парсер пишутся на С++.
	1.5. Код, написанный на языке, должен быть предельно понятным без глубокого изучения языка и при этом предельно кратким - этим язык превосходит язык Julia.
	1.6. Каждая высокоуровневая конструкция должна транслироваться в максимально понятный LLVM IR.
	1.7. Заголовочных файлов нет, каждый файл содержит один модуль.
	1.8. Полноценный ООП с перегрузкой операторов.
	1.9. В качестве операторов может быть определён любой символ из таблицы Unicode.
2. Синтаксис.
	2.1. Комментарии.
		2.1.1. Строчный комментарий - комментарий в окончании строки обозначается как в Си/Си++: 
			some_code; //комментарий...
		2.1.2. Многострочный комментарий.
			2.1.2.1. Опциональное указание названия закомментированного блока при открытии блока и при его закрытии:
				Пример 1:
					/*номер
					//..блок закоменнтированного кода...
					*/номер
				Пример 2:
					/*1
						a: vector<string>;
						/*commented_block_name
						b: string;
						*/commented_block_name
					*/1
	2.2. Формат объявления переменных:
		a = 100; // Без типа.
		a : int32; // Без инициализации.
		a : int32 = 100; // С типом и инициализацией.
		a : some_class(1, 2, is_parallel=false); //с вызовом конструктора класса с заданными параметрами.
	2.3. Система типов.
		2.3.1. Опциональная типизация. Если тип переменной не указан явно - он должен быть определён автоматически.
		2.3.2. Логический тип bool со значениями true, false.
		2.3.3. Типы целочисленных чисел.
			2.3.3.1. Наряду с int (который означает 32-разрядное знаковое целое) использовать типы int4, int8, int16, int32, int64.
		2.3.4. Типы дробных чисел.
			2.3.4.1. Числа с плавающей точкой.
				2.3.4.1.1. Числа с плавающей точкой одинарной тончости - "single" (32-bit IEEE 754 float)
				2.3.4.1.2. Числа с плавающей точкой двойной тончости - "double" (64-bit IEEE 754 double).
				2.3.4.1.3. Специальные значения: nan и inf, -inf.
				2.3.4.1.4. Основные константы: pi, pi2, exp.
				2.3.4.1.5. Результат деления любого числа на ноль равен nan и не является ошибкой выполнения.
			2.3.4.2. Дробно-рациональный тип "rational".
				2.3.4.2.1. Конструктор: rational(<числитель>,<знаменатель>):
					a: rational(1,3); // Дробь 1/3.
					b: single = 0.5;
					c = a+b; // c = 0.833333;
					c==="single"; // true.
		2.3.5. Комплексные числа "complex".
		2.3.6. Символьный тип "char".
			2.3.6.1. Из набора переменных типа "char" можно создавать вектора и матрицы.
			2.3.6.2. Вектор-строку из char можно преобразовать в string:
				a: vector<char> = ['a','b','c'];
				s: string;
				s = a; // s == ["abc"];
							2.3.6.3. Матрицу из char можно преобразовать в collection (вектор-столбец) из string:
				a: matrix<char> = ['a','b','c',,'d','e','f'];
				ss: vector<string>;
				ss = a; // ss == ["abc",, "def"];
							2.3.6.4. collection из char можно преобразовать в collection (вектор-столбец) из string:
				b: collection = ['a', 'b',, 'd', 'e', 'f'];
				ss = b; // ss == ["ab",,"def"];
		2.3.7. Строковый тип "string".
			2.3.7.1. Является классом.
			2.3.7.2 Строка является вектором-строкой из элементов типа "char".
			2.3.7.3. Методы "string".
				2.3.7.3.1. Конструкторы из числовых значений.
				2.3.7.3.2. Конструктор из массива символов.
				2.3.7.3.3. Получение символа.
				2.3.7.3.4. Присвоение символа.
		2.3.8. Время "time".
			2.3.8.1. Является классом.
			2.3.8.2. Обозначает точное время относительно стандартного момента отсчёта, так и в стандартном или заданном формате.
			2.3.8.3. Позволяет задавать и получать значений время в int64.
			2.3.8.4. Перегруженный конструктор времени, примеры:
				t: time("18:00:00.345 13.09.2024");
				t: time("180000 345 130924", "hhmmss uuu DDMMYY");
				t2: time(4677356);
			2.3.8.5. Форматированный вывод. Пример:
				print(t.formatted());
		2.3.9. Вектора обозначаются типом vector<T>, где Т - имя класса.
			2.3.9.1. Вектора индексируются, начиная с единичного индекса.
			2.3.9.2. Добавим типы для точек: point2, point3, point4 (цифра обозначает размерность).
			2.3.9.3. Добавим типы для матриц: matrix22, matrix33, matrix44, matrix34 (первая цифра - число строк, вторая - число столбцов).
		2.3.10. Матрицы обозначаются типом matrix<T>, где Т - имя класса.
			2.3.10.1. Матрицы индексируются, начиная с единичного индекса.
		2.3.11. Тип "collection" для хранения в таблице разнообразных данных. 
			2.3.11.1. Коллекции индексируются, начиная с единичного индекса.
			2.3.11.2. У этого типа есть отдельная строка-заголовок и столбец-заголовок, 
			2.3.11.3. Для этого типа есть функции чтения из файла и сохранения в файл. 
			2.3.11.4. Любой элемент коллекции также может иметь тип collection.
			2.3.11.5. Обращение к элементам коллекции ведётся через указание номера элемента:
		2.3.12. Тип слоя ("layer").
			2.3.12.1. Состоит из набора матриц одинаковой размерности.
			2.3.12.1. Конкатенация слоёв записывается в виде "[a,,, b]".
		2.3.13. Тип type обозначает тип.
			2.3.13.1 Для получения типа переменной используется функция type().
		2.3.14. Векторно-матричные операции.
			2.3.14.1. Конкатенация столбцов записывается в виде "[a, b]",
			2.3.14.2. Конкатенация строк - в виде "[a,, b]".
				Пример:
				a.header = ["Name", "Age", "City"];
							2.3.14.3. Конкатенация слоёв - в виде "[a,,, b]".
				Пример:
				a : collection = [
					"Smith", "18", "Paris",, 
					"Bob", "38", "Moscow"
				];
			2.3.14.4. Операторы нового столбца, новой строки и нового слоя перед символом окончания определения коллекции не считаются ошибкой и пропускаются при компиляции для удобства многострочной записи.
			2.3.14.5. Вектор создаётся с помощью оператора ":", например: a = 10:100.  Транспонирование - с помощью оператора "'", например: a = a';
			2.3.14.6. При многомерной индексации, оператор ":" означает, что нужно взять последовательность элементов в данной размерности:
				Примеры:
					Пример 1:
						m : matrix<single> = [1,2,3,4,,3,4,5,6]; //формирование исходного массива.
						a : vector<single> = m[1, :]; //взять все элементы первой строки;
						b : vector<single> = m[1, 2:end]; //взять элементы первой строки - со второго по последний;
						b : vector<single> = m[1, 2:2:end]; //взять элементы первой строки - каждый второй, начиная со второго;
					Пример 2:
						a.header = ["Name", "Age", "City"];
						a : collection = [
							"Smith", "18", "Paris",, 
							"Bob", "38", "Moscow"
						];
						print("Mr %s, %3i years old, lives in %s.\n", a[1,:]);
			2.3.14.7. При одномерной индексации, символ ":" преобразует матрицу в вектор-столбец. Например:
				m: matrix<double>;
				v: vector<double> = v[:];
			2.3.14.8. Для поэлементного умножения вектора столбца на вектор строку или поэлементного умножения матриц используется перед арифметическим оператором ставится символ ".".
			2.3.14.9. При несоответствии размерностей векторно-матричных операндов, в runtime осуществляется автоматическое распространие строк, столбцов, слоёв.
			2.3.14.10. Линейная индексация матриц осуществляется в направлении столбцов:
				m: matrix<single> = [1,2,3,,4,5,6,,7,8,9];
				a: single = m[4]; // Получить элемент m[2,2];
			2.3.15. Преобразование типов производится в стиле языка Си:
				a: double = 1.1432423;
				b: single = (single) a;
			2.3.16 Тип "void" обозначает отсутствие заданного типа.
			2.3.17. Получение информации о тиах во время выполнения программы (RTTI).
				2.3.17.1. Получение RTTI объекта получается вызовом конструктора класса rtti:
					type_info : rtti = rtti(some_object);
					object_name: string = type_info.name; // имя объекта
					object_fields: collection = type_info.fields; //набор значений в формате [field_name_1, field_type_1, field_value_1,, field_name_2, field_type_2, field_value_2,, ... ]
				2.3.17.2. Отображение диалога настройки объекта осуществляется вызовом метода "show" класса rtti:
					type_info : rtti = rtti(some_object);
					some_object_edited = type_info.show();
			2.3.18. Тип "method" имеют переменные, обозначающие функции и методы классов/интерфейсов.
			2.3.19. Для преобразования пользовательских типов используются интерфейсы:
				interface class Shape
					name: string;
					position: Point2;
					
					some_common_actuion(a: int32, b: single) : auto;
				end
				
				some_variable_1: Circle;
				some_variable_2: Rectangle;
				some_variable_3: matrix<Shape> = [some_variable_1, some_variable_2];
			2.3.20. Переменные могут одновременно принадлежать нескольким типам, при присвоении совершается попытка преобразования к одному из указанных типов, например:
				some_variable: [bool, int32];
	2.4. Повторное использование кода.
		2.4.1. Функции и методы.
			2.4.1.1. Методы находятся внутри классов и интерфейсов, функции находятся снаружи классов и интерфейсов.
			2.4.1.2. Использовать стиль Matlab для блоков кода: Фигурные скобки "{" в начале блоков не ставятся, в конце блоков ставится end.
			2.4.1.3. Формат функций:
				some_function(a, b, c=10, d=20) : int32
					//...
					return true; //успешное выполнение.
				end
			2.4.1.4. На входе каждой функции или метода есть как обязательные аргументы, так и опциональные, например:
				some_function(a, b : single, is_parallel=true, n=100) : int
					// Тело функции.
				end
			2.4.1.5. Если тип не указан, но функция должна возвращать значение, то использовать auto:
				some_function(a, b, c=10, d=20) : auto
					//...
					return true; //успешное выполнение.
				end
			2.4.1.6. Если функция ничего не возвращает, то использовать "void" или "[]":
				some_function(a, b, c=10, d=20) : void
					//...
					return; //Если требуется досрочный выход из функции.
					//...
				end
			2.4.1.7 На выходе функция может выдавать несколько значений, каждое из которых может быть строго именовано:
				some_function(a, b) : [new_time: double, bool]
					//...тело функции ...
				end
			2.4.1.8. для получения логического значения присвоенности результата выполнения функции некоторой переменной используется ключевое слово "assigned". Например:
				some_function(a, b) : bool
					if !assigned
						print("Значение не присвоено.\n");
					end
				end
			2.4.1.9. Как у Matlab, в конце списка аргументов можно указать структуру, поля которой заполняются внутри конструкции arguments, после обязательных полей, для валидации значений аргументов используется конструкция вида {@some_validation_function_1, @some_validation_function_2}, например:
					some_function(a, b, options : collection) : void
						arguments
							a {@islogical}; 					// Обязательное поле.
							b {@islogical}; 					// Обязательное поле.
							options.is_parallel = true; 		// Опция.
							options.count {@ispositive} = 100; 	// Опция.
						end
					end
				Пример вызова такой функции:
					some_function(true, false, count=53); //аргументу "is_parallel" присвоено значение по умолчанию.
				Либо:
					options : collection = ["count", 53,, "is_parallel", true];
					some_function(true, false, options);
			2.4.1.10. Стандартные методы валидации:

		2.4.3. Классы.
			2.4.3.1. Формат класса:
				template<T>
				class Shape
				public:
					constructor() 
						// операции, выполняемые сразу после создания объекта.
						string = "Unnamed Shape";
					end
					
					destructor()
						//операции перед уничтожением объекта и освобожденииюем памяти.
					end
					
				protected:
					name: string;
					position: point2<T>(0, 0);
					//...
				private:
					//...
				end
				
				template<T>
				class Circle: public Shape<T>
				public:
					get_radius() : T
						return radius;
					end
				protected:
					radius: T = 1;
				end
				
				class Rectangle: public Shape
				public:
					width: single = 1;
					height: single = 1;
				end

				private:
					//эти объявления доступны только внутри этого класса.
				protected:
					//эти объявления доступны только внутри этого класса и внутри его потомков.
				end
			2.4.3.3. Для обращения к внутренним полям класса использовать ".". Например: .c = .a + .b;
			2.4.3.4. Как в С++, методы класса можно перегружать.
			2.4.3.5. Метод является полем класса.

	2.10. Операции
		2.10.1. Логические операции:
			2.10.1.1. "!" - логическое "нет", например: !false==true;
			2.10.1.2. "!=" - логическое "не равно", например: false!=true;
			2.10.1.3. "==" - логическое "равно", например: true==true;
			2.10.1.4. "===" - сравнение типов (значения при этом не сравниваются!), например:  true===false; 
			2.10.1.5. "&" - логическое "и";

			2.10.1.9. "<" - логическое "меньше";
			2.10.1.10. ">=" - логическое "больше или равно";
			2.10.1.11. "<=" - логическое "меньше или равно";

			2.10.1.14. ".&" - поэлементное "и";
			2.10.1.15. ".|" - поэлементное "или";

			2.10.1.18. ".>=" - поэлементное "больше или равно";

				get_metrics() : [metrics: collection]; // вывод состояния объекта в консоль (если нет присвоения результата выполнения метода) или в память (при присвоении результата выполнения метода некоторой переменной).

6. Ускорение вычислений.
	6.1. Внутри языка предусмотрены средства для ускорения вычислений, в т.ч. векторно-матричных операций, с помощью специализированных устройств, для примера реализована встроенная полнофункциональная поддержка CUDA, SSE, AVX, AVX2, AVX512, и расширений для ARM.
	6.2. Для объявления ядра для запуска на GPU нужно использовать конструкции. Примеры:
		Пример 1:
			config : parallel(grid=32, block=256); //config - название переменной, хранящей экземпляр класса parallel.
			parallel(config) auto* vector_add(a, b); // в auto хранится результат операции - указатель на данные внтури памяти GPU.
		Пример 2:
			parallel(grid=32, block=256) single* vector_add(a, b); // в single* хранится результат операции - указатель на данные внтури памяти GPU.
		Если на CPU, то:
			A: vector<double>; //массив, элементы которого будут заполняться параллельно.
			parallel(threads=8, array=&A) for i in 1:4:1000 //цикл от 1 до 1000 с шагом 2, вместо вектора может быть подставлен некоторый итерируемый объект.
				A[i] = i;
			end
	6.3. Данные с GPU можно передать в память хоста с помощью вызова метода host_variable, при этом осуществляется копирование из данных по указателю "some_gpu_variable" в переменную "host_variable" заданного типа:
		auto = parallel.get(some_gpu_variable);  //хост-переменная имеет тип "auto".
	6.4. Данные с хоста на GPU передаются через вызов "parallel.set()", например:
		gpu_variable: auto = parallel.set(some_host_variable);
	6.5. Если при вызове функции на GPU переменная хранится в памяти хоста, то она передаётся на GPU автоматически.
	6.6. Освобождение памяти переменными, хранящимися на GPU, выполняется с помощью ключевого слова "clear":
		clear(gpu_variable);
7. Разработка с помощью LLM-моделей.
	7.1. Синтаксис языка должен быть легко понятен LLM-моделям: избегать неявных импортов и преобразований имён переменных, стремиться к лаконичному синтаксису (избегать "синтаксического сахара" и символизации).
	7.2. Блок кода, для которого запрашивается ответ от LLM, выделяется ключевыми словами #intent ... #end, например:
		reading_function(file: string) : [out: string]
			#intent Прочитать текст из файла на пути file и вернуть прочитанный текст в "out"
			#generated
				// сгенерированный текст.
			#end
		end
	7.3. Блок кода, который не может быть изменён при получении ответа от LLM, выделяется ключевыми словами #fixed ... #end, например:
		draw_matrix(m : matrix<double>) : void
		#intent нарисовать матрицу и нарисовать точку в центре матрицы.
		#generated
			//...
			#fixed
				a: double = 0.0;
			#end
			//...
		#end
