# Совместное редактирование текста в Centauri IDE

## Обзор

Реализована система совместного редактирования кода в реальном времени для IDE Centauri. 
Система позволяет нескольким разработчикам работать над одним проектом одновременно, 
видя изменения друг друга в реальном времени.

## Компоненты

### 1. CollaborationService (`src/services/collaboration/`)

Базовый сервис для управления сессиями совместной работы:

- **Хостинг сессий**: Создание и управление сессиями через TCP-сервер
- **Подключение к сессиям**: Клиентское подключение к существующим сессиям
- **Синхронизация документов**: Отправка и получение изменений файлов
- **Управление курсорами**: Передача позиций курсоров между участниками
- **Выделения текста**: Синхронизация выделенных областей
- **Блокировка файлов**: Предотвращение конфликтов редактирования
- **История изменений**: Хранение и откат изменений
- **Аутентификация**: Регистрация и вход пользователей

### 2. RemoteCursorOverlay (`src/ui/editor/RemoteCursorOverlay.*`)

Виджет для отображения удалённых курсоров и выделений:

- **Отрисовка курсоров**: Цветные вертикальные линии с метками имён
- **Полупрозрачные выделения**: Подсветка текста, выделенного другими участниками
- **Прозрачность для мыши**: Не мешает взаимодействию с редактором
- **Автоматическое обновление**: Перерисовка при изменении позиций

### 3. CollaborativeTextManager (`src/ui/editor/CollaborativeTextManager.*`)

Менеджер интеграции сервиса с редактором кода:

- **Отслеживание изменений**: Автоматическая отправка локальных изменений
- **Применение удалённых изменений**: Корректное обновление текста
- **Синхронизация курсора**: Передача позиции курсора в реальном времени
- **Предотвращение циклов**: Флаг для избежания бесконечных обновлений
- **Конвертация позиций**: Преобразование между позициями документа и строками/колонками

### 4. CollaborationDialog (`src/ui/dialogs/CollaborationDialog.*`)

Пользовательский интерфейс для управления сессиями:

- **Создание сессии**: Запуск новой сессии совместной работы
- **Поиск и подключение**: Список доступных сессий
- **Чат**: Обмен сообщениями между участниками
- **Список участников**: Отображение подключённых пользователей
- **Статус подключения**: Индикация состояния соединения

## Использование

### Начало работы

```cpp
// В MainWindow или другом месте инициализации
auto* collaborationService = new CollaborationService(this);
auto* collaborativeManager = new CollaborativeTextManager(codeEditor, this);

// Установить сервис в менеджер
collaborativeManager->setCollaborationService(collaborationService);

// Начать отслеживание файла
collaborativeManager->startTracking("path/to/file.prx");
```

### Хостинг сессии

```cpp
// Зарегистрировать пользователя
collaborationService->registerUser("username", "email@example.com", "password");
collaborationService->login("username", "password");

// Создать сессию
if (collaborationService->hostSession("MyProject", 8080)) {
    qDebug() << "Сессия создана, ID:" << collaborationService->getSessionId();
}
```

### Подключение к сессии

```cpp
// Присоединиться к существующей сессии
if (collaborationService->joinSession("192.168.1.100", 8080, "password")) {
    qDebug() << "Подключено к сессии";
    collaborativeManager->startTracking("file.prx");
}
```

### Обработка событий

```cpp
// Подключение к сигналам
connect(collaborationService, &CollaborationService::changeReceived,
        [](const DocumentChange& change) {
            qDebug() << "Получено изменение от" << change.userId;
        });

connect(collaborativeManager, &CollaborativeTextManager::remoteCursorUpdated,
        [](const QString& userId, int line, int column) {
            qDebug() << "Курсор пользователя" << userId << "на позиции" << line << ":" << column;
        });
```

## Протокол обмена

Сообщения передаются в формате JSON через TCP:

```json
{
  "type": "change",
  "session_id": "...",
  "user_id": "...",
  "username": "...",
  "timestamp": 1234567890,
  "change_id": 1,
  "file": "path/to/file.prx",
  "start_line": 10,
  "end_line": 15,
  "old_content": "...",
  "new_content": "...",
  "version": 42
}
```

Типы сообщений:
- `change` - изменение содержимого файла
- `cursor` - обновление позиции курсора
- `selection` - обновление выделения
- `sync_request` - запрос полной синхронизации
- `file_lock` / `file_unlock` - блокировка файла
- `auth` / `auth_response` - аутентификация
- `keepalive` - поддержание соединения

## Особенности реализации

### 1. Версионирование
Каждое изменение получает уникальный номер версии, что позволяет:
- Отслеживать историю изменений
- Выполнять откат к предыдущим версиям
- Обнаруживать конфликсты

### 2. Блокировка файлов
Для предотвращения конфликтов реализована система блокировок:
- Только один пользователь может редактировать файл одновременно
- Другие видят файл в режиме только для чтения
- Блокировка автоматически снимается при закрытии файла

### 3. Оптимизация трафика
- Изменения отправляются пакетно с небольшой задержкой
- Курсоры обновляются с ограниченной частотой
- Используются дельта-изменения вместо полной пересылки

### 4. Безопасность
- SHA-256 хеширование паролей
- Опциональное шифрование соединений
- Проверка подлинности участников
