Давай разработаем IDE для разрабатываемого языка программирования Proxima. Внимательно изучи описанные требования и предложения, глубоко подумай и напиши развёрнутый дизайн-документ для создания данной IDE.

Особенности IDE:
1. Название IDE: Centauri.
2. IDE написана на Qt6/C++.
3. Поддержка языка Proxima.
4. Каждый проект хранится в отдельной папке.
5. Исходные тексты языка ("модули") имеют расширение ".prx" и хранятся в папке "src", в которой могут создаваться папки для подпроектов.
6. Сборочный файл проекта называется "build.rules" и хранится в папке "rules".
7. Файл с описанием специфики проекта называется "design.rules" и хранится в папке "rules".
8. Файл с описанием правил оформления кода проекта называется "code.rules" и хранится в папке "rules".
9. Файл с описанием требований к интерфейсу пользователя проекта называется "ui.rules" и хранится в папке "rules".
10. В IDE можно редактировать множество исходных текстов модулей, исходные тексты отображаются в окне редактора, 
11. Окно редактора можно дублировать, чтобы одновременно видеть несколько исходных текстов или один и тот же исходный текст в разных местах. Дополнительные окна редактора располагаются вне окна приложения.
12. IDE позволяет быстро выбрать редактируемый модуль с помощью табов, поиска, стрелок навигации по последним файлам, шорткатов, выбора подпроекта, дерева файлов.
13. Табы выбора редактируемого модуля отображаются вдоль левого края каждого окна редактора и ориентированы горизонтально.
14. IDE содержит средства отладки: создание безусловной точки останова, точки останова по условию.
15. IDE содержит инструменты визуализации состояний объектов, которые при остановке/бездействии/завершении программы позволяют изменять значения:
- для class - список полей.
- для vector - график, который можно приближать, удалять вокруг положения указателя, двигать, комбинировать с другими графиками и наносить на них отдельные точки, смотреть значение в точке, ближайшей к указателю (дополнительные инструменты - отображение пиков и впадин, выделение диапазона и копирование его данных, и др.);
- для matrix - либо таблица, либо изображение с настраиваемой палитрой, которое можно приближать, удалять вокруг положения указателя, двигать, на изображение можно накладывать изображения других матриц, графики, точки, можно смотреть значение в ячейке, ближайшей к указателю, при большом приближении в каждой ячейке отображается соответствующее значение матрицы, дополнительные инструменты - выделение диапазона и копирование его данных, обрезка матрицы, и др.
- для layer - 3D визуализация с возможностью выбора сечения, которую можно приближать, удалять вокруг положения указателя, двигать, величина значения в ячейке матрицы слоя определяет прозрачность вокселя дополнительные инструменты - выделение диапазона и копирование его данных, и др.).
- для collection - таблица с заголовками столбцов и строк.
16. В текстовом редакторе каждого модуля слева (правее табов) отображается панель инструментов, на которой содержатся пиктографические кнопки:
а) кнопки контроля выполнения (первая иконка - пуск/пауза, вторая иконка - пуск до строчки выхода из метода, третья иконка - прекращение выполнения);
б) кнопка визуализации значений переменной, на которой стоит курсор;
в) кнопка автоформатирования исходного текста;
г) кнопка вставки сниппета;
д) кнопка вызова контекстного меню помощи в оформлении комментариев и написании формул:
- форматирование текста;
- математическая разметка.
е) кнопка переключения режима отображения текстового редактора:
- стандартный с подсветкой синтаксиса;
- подсветка типов переменных, автоматически выбранных при компиляции (на момент остановки/бездействия/завершения программы) - в индексных цветах:
	int,int32,int16,int8,int4 - оранжевый;
	int64 - красный;
	single - голубой;
	double - синий;
	vector - розовый;
	matrix - тёмно-розовый;
	collection - фиолетовый;
	char - зелёный;
	string - тёмно-зелёный;
- подсветка размерности переменных, автоматически выбранных при компиляции (на момент остановки/бездействия/завершения программы) - в индексных цветах:
	скаляр - фиолетовый
	от 2 до 12 - синий;
	от 13 до 64 - голубой;
	от 65 до 16,384 - зелёный;
	от 16,385 до 5,000,000 - оранжевый;
	от 5,000,000 - красный.
- подсветка времени, потраченного на выполнение кода - на момент остановки/бездействия/завершения программы - в цветах от холодного к горячему;
- подсветка частоты выполнения участка кода - на момент остановки/бездействия/завершения программы - в цветах от холодного к горячему;
- выделение цветом внесённых изменений по сравнению с файлом, хранящимся в репозитории GIT в выбранной ветке;
- автор правки - отображается случайным цветом, присвоенным при регистрации автора на хосте, цвет выбирается максимально отличным от уже зарегистрированных;
- старость правки - отображается цветом в зависимости от времени, прошедшего с момента внесения правки (минуты, часы, дни, недели, годы) - в индексных цветах, от холодного к горячему.
Для мониторинга старости правки, каждый исправленный блок хранится в папке  ".stories/" в файле "*.story" c соответсвующем названием в формате collection:
[Автор_правки, вид_изменения, дата_изменения,, ...];
при активации данного режима производится парсинг истории из файла ".story" и отображаются соответствующие цвета.

17. Внизу текстового редактора отображается строка состояния, в которой отображается:
- позиция курсора в файле;
- статус выполнения, позиция выполнения;
- статус сохранения изменения;
- текущее системное время - отображается только в активном окне.
- ресурсы системы (ЦПУ, память, диск) - отображается только в активном окне.
- состояние подключения к LLM-модели:
	название модели;
	размер свободного контекста и полный размер контекста;
	статус генерации ("gen" - формирование нового кода, "fix" - исправление ошибок, "doc" - формирование документации, "error" - ошибка, "idle" - простаивание).
18. В IDE есть консоль, в которой отображаются в отдельных табах:
- процесс сборки;
- ошибки сборки приложения;
- стандартный вывод приложения;
- вывод ошибок приложения;
- вывод работы LLM-модели;
- call-stack со значениями параметров.
19. В IDE интегрировано взаимодействие с GIT.
20. Каждые 5 минут редактируемый файл автосохраняется в папке "autosave" по пути, соответствующему его пути в папке "src".
21. Если файл не редактировался более 5 минут, его копия однократно сохранятется в папке "autosave" по пути, соответствующему его пути в папке "src", и больше автосохранений этого файла не происходит до следующего редактирования.
22. Пиктографическое главное меню приложения: "Файл", "Генерация", "Настройки", "Помощь" (в компактном виде, без мультидокументного поля и без строки состояния), в которых
а) "Файл" - создание, удаление, сохранение файла, сохранение всех файлов, создание проекта, открытие проекта, выход;
б) "Проект" - настройки проекта;
в) "Выполнение" - управление выполнением (в т.ч, уровень отладки (англ.:verbose-level) - подсказывает рантайму, какие отладочные сообщения писать в консоль, ещё можно установить максимальный объём выделяемой памяти и разрешённые наборы команд и устройств);
г) "Генерация" - окно запроса к LLM для помощи в ИИ-генерации кода, подпункты меню:
- "ручная" - генерация для каждого intent с поблочным применением правок;
- "авто" - автоматическая генерация для каждого intent;
- "документация" - генерация документации для проекта.
д) "Настройки" - настрока параметров IDE и времени выполнения:
- число выделенных ядер процессора;
- объём выделенной оперативной памяти;
- объём выделенного места на жёстком диске;
- доступ приложения к файлам вне папки проекта.
е) "Сведения":
- "сведения об активных пользователях" - имена пользователей и время последней правки.
- "Сведения о сборке" - диалоговое окно для формирования отчёта о сборке (md5 hash, дата сборки, название сборки, номер версии, имена участников с указанием даты последней правки, ...);
- "Сведения о совместимости" - Построение отчёта о совместимости кода проекта с текущей версией компилятора;
ж) "Помощь" - в нём содержатся подпункты помощи:
- "Помощь по IDE" показывает в гипертекстовом формате руководство по использованию IDE;
- "Помощь по Proxima" показывает в гипертекстовом формате руководство по использованию языка Proxima;
- "Помощь по сборке" показывает в гипертекстовом формате руководство по настройке и сборке проекта;
- "Помощь по проекту" показывает в гипертекстовом формате (с поддержкой отображения формул) руководство, сформированное при запуске сборки с аргументом "doc" на основе исходных текстов проекта и комментариев к нему.
- "О программе".
- "Об авторах".
23. В запущенном экземпляре IDE ведётся работа только с одним проектом.
24. Код IDE предельно простой, интуитивно понятный, хорошо задокументирован.
25. IDE работает в режимах:
- Режим редактирования (Edit Mode);
- Режим выполнения (Runtime Mode);
- Режим паузы (Pause Mode) - временной остановки выполнения.
26. Схемы запуска программы на выполнение:
- Режим релиза;
- Режим отладки;
- Режим отладки - выполнение до точки останова;
- Режим отладки - выполнение до курсора;
- Режим отладки - выполнение до последней инструкции метода;
- Режим отладки - выполнение до первого метода.
27. В режиме релиза IDE создаёт минимальную нагрузку на процессор.
28. Лицензия GPLv3.
29. IDE должна уметь обращаться через API к LLM, выполняющейся на сервере LM-Studio, и выдавать предложения по модификации кода в рамках #intent, и пользователь может поблочно выбирать (подобно GIT), какие модификации следует применить к коду.
30. IDE должна поддерживать одновременное многопользовательское редактирование документов внутри локальной сети и через интернет с регистрацией на хосте с хранением паролей в зашифрованном виде и авторизацией.
31. Должен вестись учёт правок пользователей с возможностью их частичного или полного отката хостом.
32. Поддержка сворачивания тела методов и классов, а также блочных комментариев вида:
/*название
//тело комментария
*/название
А также регионов кода:
#region название
//код...
#endregion название
33. В состав IDE  входит пакетный менеджер, позволяющий загружать пакеты из указанного в настройках репозитория и отправлять запрос на регистрацию текущей версии разрабатывамого пакета в репозитории.
34. IDE хранит спецификацию языка Proxima в векторизованном виде.
35. При запросе к LLM, IDE передаёт необходимый контекст:
- спецификацию языка Proxima;
- правила оформления кода;
- спецификацию проекта;
- объявления и исходные тексты подключенных файлов;
- intent.
36. При ответе LLM в ручном режиме, IDE накладывает ответ поверх кода (включая зоны фиксированного кода), позволяя применить изменения поблочно.
37. При ответе LLM в автоматическом режиме, IDE полностью применяет новый код (кроме зон зафиксированного кода) и запускает тесты. При наличии ошибок IDE делает новый запрос к LLM с указанием кодов ошибок. Коды ошибок должны быть прописаны в спецификации языка.
